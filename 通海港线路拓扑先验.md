# 通海港线路拓扑先验

## 结论

通海港物流基地的最终轨道中心线不应被理解成一堆平等的线段，而应先按站场拓扑分层：

```text
一条贯通主线
  + 最多两条局部并行侧线 / 装卸线 / 停留线
  + 若干道岔或渡线平滑连接这些平行股道
```

用户提供的站场示意图只用于表达拓扑关系，不能按比例、折线角度或位置直接套到真实 DOM 坐标上。

## 真实结构假设

- 正中间存在一条近似直线的贯通主线。
- 这条主线从港口一端贯穿到另一端，经过道岔区时不应中断。
- 同一横断面最多约三条主要平行轨道中心线。
- 除贯通主线外，另外一到两条并行线可以有真实首尾端点。
- 侧线、装卸线、调机停留线等允许是死路，不应为了降低 dead-end 数量而强行接到别的轨道上。
- 平行轨道之间通过道岔或渡线连接。
- 道岔连接应与被连接轨道位置连续、方向连续，至少在连接起点附近近似相切。

## 算法原则

后续中心线构建应采用“主线优先”的顺序，而不是继续把自动线、人工 hint、QGIS 修正线全部平等吸附：

1. 先识别贯通主线 `main_through_track`。
2. 对主线施加强约束：贯穿全场、近似直线、道岔区域不中断。
3. 再识别上/下局部并行侧线，允许它们在合理位置终止。
4. 最后识别道岔/渡线连接，并把它们作为连接边挂接到主线和侧线上。
5. 只修复主线中段的断点；不要把所有 dead-end 都当成错误。
6. 对道岔连接做方向约束，避免横切、硬折、错接到相邻股道。

## 当前 v6 结果的风险

`output/rail_centerline_final_v6_qa_bridge/final_centerline_network.shp` 已经能作为当前最好的候选线网查看，但它的构建逻辑仍偏向通用网络合并：

- QGIS accepted geometry 会覆盖附近自动/hint/feedback 线。
- 端点会吸附到附近端点或线段内部。
- 重复线段会被删除。
- QA bridge 会作为额外线段加入。

这些规则能修复很多局部断裂，但还没有显式表达：

- 哪一条是贯通主线；
- 哪些线是侧线或终端线；
- 哪些 dead-end 是真实侧线端点；
- 哪些道岔连接必须与主线相切。

因此，道岔区域效果差时，不应只继续调吸附距离或去重阈值，而应先引入站场拓扑先验。

## DOM 叠图核查要求

每次生成最终中心线候选后，应至少做以下视觉核查：

- 把中心线叠加到 DOM 上，不看裸 Shapefile。
- 优先裁切道岔/渡线密集区、端点密集区和主线疑似断点区。
- 检查主线是否落在正中间那条轨道中心附近，并且是否贯穿道岔。
- 检查侧线死端是否合理，不要因为有端点就强行连接。
- 检查道岔支线是否从主线或侧线平滑分出，而不是横切或跳到相邻股道。

## 下一步实现方向

第一版实验可以先做诊断而不是直接替换最终成果：

```text
final_centerline_network
  -> DOM 裁切叠图
  -> 识别线段来源和端点/高阶节点
  -> 按主方向投影为 s/t 坐标
  -> 聚类出最多三条平行轨道骨架
  -> 标记 main / siding / connector / terminal
  -> 只对 main 的中段断点提出桥接或重建建议
```

如果这一步确认主线和侧线分层合理，再把它并入最终构建脚本。

QGIS 验收时不要把 proposal 直接当成最终中心线。优先看字段：

- `geom_kind`：`raw_transition_evidence` 是真实候选碎线，`evidence_curve_proposal` 是沿 evidence 曲率生成的候选曲线。
- `pair_id`：连接的股道带，当前只做 `minus_to_main` 和 `main_to_plus`。
- `connector_score`：粗略可信度，低分短候选必须更谨慎。
- `completion_m`：proposal 相对 raw evidence 的短补全长度；无锚点时默认不再强制外推。
- `anchor_id`：如果使用了人工转辙机锚点，会记录锚点编号。
- `split_id` / `straight_tail_m`：如果用户指出某坐标之后已经是直轨，connector 会在该点裁切，裁掉的直轨长度写入 `straight_tail_m`。

- `review_status`：当前全部为 `candidate_needs_dom_review`，还没有进入最终拓扑。

这一版的意义不是自动补齐所有道岔，而是把“确实有斜向过渡证据的位置”先挑出来，并尽量保持线条压在 DOM 可见轨道中心。通过 QGIS/DOM 复核之前，最终线网仍应只保留已验收的 2股道主线和平行股道骨架。

### 2026-05-20 手扫 LAS 反推钢轨验证

用户在 QGIS 中复核 `turnout_crossover_connector_proposals.shp` 后确认：`CX01`、`CX02` 已经可用并落在轨道内部，但不完全压在几何中心线上。为验证偏差是否能由点云纠正，新增脚本把候选中心线按轨距反推出左右钢轨，再用手扫 LAS 中的轨头高度点云脊线反推真实钢轨对中心：


主要产物：

- `turnout_crossover_predicted_rails.shp`：由当前中心线和 `1.6 m` 轨距反推出的两根钢轨线。
- `turnout_crossover_las_observed_rails.shp`：LAS 轨头点云峰值修正后的两根钢轨线。
