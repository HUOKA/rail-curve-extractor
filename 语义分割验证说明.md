# 语义分割验证说明

## 结论

语义分割明显更适合当前标注方式。你现在的 `track_area` 实际是“可见钢轨本体”，目标细、长、连续，YOLO instance segmentation 容易碎片化和漏检；轻量 U-Net 语义分割直接学习每个像素是不是钢轨，当前验证效果更好。

## 数据

- 数据集：`data/datasets/rail_seg_v2_track_only`
- 图像总数：56 张
- 有钢轨标注图：31 张
- `track_area` 实例：220 个
- 未标注图：25 张，仍按未知样本处理，不作为确认负样本

训练拆分：

| 项目 | 数量 |
| --- | ---: |
| 源图训练 | 25 |
| 源图验证 | 6 |
| 训练 crop | 125 |
| 验证 crop | 30 |

## 方法

本轮不是 YOLO，而是真正的二值语义分割：

```text
CVAT polygon
  -> 栅格化为 rail/background mask
  -> 轻量 U-Net 训练
  -> 输出钢轨概率图
  -> 阈值化为钢轨 mask
  -> 后续可骨架化、配对左右钢轨、求中心线
```

训练环境仍是 `.yolo-venv`，没有污染主 `.venv`。

## 当前指标

输出目录：`output/rail_seg_semantic_unet_v1`

验证 crop 最佳阈值指标：

| 指标 | 数值 |
| --- | ---: |
| Precision | 0.9173 |
| Recall | 0.9425 |
| F1 | 0.9297 |
| IoU | 0.8687 |
| Best threshold | 0.85 |

整张验证图重新计算：

| 指标 | 数值 |
| --- | ---: |
| Precision | 0.9512 |
| Recall | 0.8900 |
| F1 | 0.9196 |
| IoU | 0.8511 |

这比 YOLO track-only 的 `Mask precision 0.7503 / recall 0.3046 / mAP50 0.3742` 更符合“先尽量找全钢轨，再做几何后处理”的需求。

## 注意

这个结果不能理解为已经泛化成功。当前训练图和验证图来自同一条 aligned DOM 走廊，相邻图像外观很像，所以指标可能偏乐观。它能说明的是：

- 你的钢轨本体 polygon 标注没有白费。
- 语义分割路线比 YOLO 更适合当前目标。
- 现在已经可以进入下一步：概率图后处理、钢轨骨架化、左右轨配对、中心线提取。

## 复现实验

```powershell
.\.yolo-venv\Scripts\python.exe .\scripts\train_rail_seg_semantic.py `
  --dataset .\data\datasets\rail_seg_v2_track_only `
  --out .\output\rail_seg_semantic_unet_v1 `
  --epochs 30 `
  --batch 2 `
  --device cuda `
  --workers 0 `
  --input-width 384 `
  --input-height 1024 `
  --tile-height 1024 `
  --tile-stride 512 `
  --base-channels 16
```
